## 系统概述

### 系统简介

采购管理系统是一款面向企业采购部门的桌面应用程序，基于 Python + CustomTkinter + SQLite 技术栈开发。系统采用莫兰迪暖色调 UI 设计，支持 Windows 10/11 原生适配，提供从物料管理、供应商管理到采购垫付、差旅报销的全流程数字化管理。

### 主要功能

| 模块 | 功能描述 |
|------|----------|
| 仪表盘 | 统计概览面板，展示各模块的关键指标 |
| 物料查询 | 物料台账查询，支持 CSV/JSON 导入、Excel 导出、多维度筛选 |
| 物料下单 | 包材/物料全流程下单管理（比价→合同→通知→生产→发货），支持 Excel 导入导出 |
| 报价单 | 产品报价单管理，阶梯定价，基于模板导出 Excel |
| 供应商管理 | 供应商信息维护，Excel 导入导出 |
| 催款记录 | 付款催收记录管理，Excel 导入导出 |
| 采购垫付 | 采购垫付款项记录，含物料明细，Excel 导入导出 |
| 差旅报销 | 差旅费用报销管理，含交通/住宿明细，Excel 导入导出 |
| 备忘录 | 工作备忘记录，支持标记完成 |
| 设置 | 外观、数据管理、启动设置、系统设置、版本信息 |

## V1.6 版本更新内容

### 物料下单页面增强

1. **新增供应商筛选**：筛选栏新增供应商文本输入框，支持实时模糊匹配（包含关系），替代原有的下拉选择
2. **新增项目号筛选**：筛选栏新增项目号输入框，限制8位纯数字，支持实时前缀匹配
3. **新增清除筛选按钮**：一键重置所有筛选条件
4. **新增下单数量字段**：在表单基本信息区域（项目号下方）新增"下单数量"输入框
5. **列表表格新增下单数量列**：表格在项目号列后新增"下单数量"显示列
6. **CSV 导入模板**：物料查询页面新增 CSV 格式导入模板附件

### Bug 修复

1. **修复 CTkComboBox 递归回调导致页面白屏**：`_refresh_project_filter()` 中 `configure(values=...)` 触发 command 回调导致无限递归，通过临时移除 command 回调解决
2. **修复 CTkEntry 不支持 variable 参数**：customtkinter 的 CTkEntry 与 tkinter 原生 Entry 不同，不支持 `variable=StringVar`，改用 `entry.get()/delete()/insert()` 方式操作文本

### main.py 核心结构

**类 `App(ctk.CTk)`**：

- 窗口尺寸：默认 1280×800，最小 1100×700
- 单进程互斥锁（Windows Mutex）
- 全局异常日志（error.log）
- PyInstaller 资源路径兼容（`sys._MEIPASS`）

**页面路由 `_switch_page(key)`**：

| key | 页面类 | 说明 |

| dashboard | DashboardPage | 仪表盘 |
| packaging | PackagingPage | 物料下单 |

**系统托盘**：支持最小化到托盘、开机自启、窗口置顶

### database.py — 数据库模块

**类 `Database`**：封装所有数据库操作，支持自定义数据目录。

**初始化**：

db = Database(data_dir=None)  # 默认 ~/采购管理系统数据/

| `execute(sql, params=())` | 执行 INSERT/UPDATE/DELETE |
| `executemany(sql, seq_of_params)` | 批量执行 |
| `create_tables()` | 创建所有数据表 |
| `_migrate_tables()` | 数据库迁移（列名重命名、缺失列添加） |
| `export_to_xlsx(headers, rows, filepath)` | 静态方法，通用 Excel 导出（openpyxl） |

### packaging_page.py — 物料下单页面

**类 `PackagingPage(ctk.CTkFrame)`**：物料下单列表页面

**筛选功能**：
- 项目下拉（`CTkComboBox`）：精确匹配项目名称
- 供应商输入框（`CTkEntry`）：实时模糊匹配，包含关系
- 项目号输入框（`CTkEntry`）：8位纯数字，实时前缀匹配
- 清除筛选按钮：一键重置所有筛选条件

**防递归机制**：`_refresh_project_filter()` 中先 `configure(command=None)` 移除回调，更新 values 后再恢复 `configure(command=...)`

**类 `PackagingForm(ctk.CTkToplevel)`**：新增/编辑物料下单表单弹窗

表单结构：基本信息 + 五环节折叠面板
- 基本信息：物料名称、所属项目、项目号、下单数量、下单厂家
- ① 比价：单价、日期、备注
- ② 合同：状态、备注
- ③ 通知：日期、沟通货期、备注
- ④ 生产：生产周期、预定发货日期、备注
- ⑤ 发货：日期、方式、物流、预计到货、通知库房

### query_page.py — 物料查询页面

**类 `QueryPage(ctk.CTkFrame)`**：物料台账查询页面

**CSV 导入**：自动检测编码（utf-8-sig → gbk → gb2312 → utf-8），自动识别列名映射（支持中英文）

**支持识别的列名**：
| 数据库字段 | 可识别的列名 |
|-----------|-------------|
| contract_no | 合同编号、合同号、编号、contract |
| supplier | 供应商名称、供应商、厂商、vendor |
| item_no | 物料项目号、项目号、物料号、物料编码、料号 |
| material_name | 物料名称、品名、产品名 |
| quantity | 数量、qty |
| unit | 单位、计量单位 |
| unit_price | 采购单价、含税单价、单价、价格 |
| amount | 订单总额、含税金额、金额、总价、合计 |

**筛选功能**：年份、供应商（模糊）、物料名称（模糊）、项目号（精确）

## 打包说明

### 环境要求

- Python 3.12+
- CustomTkinter 5.2+
- PyInstaller 6.0+
- Pillow (PIL) 10.0+
- openpyxl 3.0+
- pystray 0.19+

### 打包步骤

1. **安装依赖**：
   ```
   pip install customtkinter Pillow openpyxl pystray pyinstaller
   ```

2. **执行打包**：
   ```
   pyinstaller --clean --noconfirm "采购助手V1.3.spec"
   ```

3. **打包完成后**，exe 文件位于 `dist/采购助手.exe`

### 注意事项

- spec 文件中 `name='采购助手'`，不显示版本号
- `console=False`，不显示控制台窗口
- `icon='assets/同仁堂企业LOGO2.ico'`，使用同仁堂企业图标
- `datas` 中包含所有资源文件（assets/ 目录下的所有文件）
- `hiddenimports` 包含 `pystray._win32` 用于系统托盘功能

### 数据目录

- 默认数据目录：`~/采购管理系统数据/`
- 数据文件：`procurement.db`（SQLite 数据库）、`settings.txt`（设置文件）、`error.log`（错误日志）
- 支持在设置页面自定义数据目录，迁移时自动复制已有数据库

**3.2 架构模式**

- 单窗口多页面：ctk.CTk 主窗口 + CTkFrame 页面切换

- 侧边栏路由：每个导航按钮绑定 \_switch_page(key)，先 destroy
  旧页面再实例化新页面

- 数据库单例：App 级别唯一 Database 实例，check_same_thread=False

- 模态表单：CTkToplevel + grab_set()，保存后回调 on_save 刷新主页面

- 资源兼容：\_get_resource_path() 处理 PyInstaller \_MEIPASS
  与源码路径双模式

- 全局异常：sys.excepthook → error.log，表单创建 try-except 弹窗

**四、数据库设计**

**4.1 数据库位置与基础信息**

数据库文件：%USERPROFILE%\\采购管理系统数据\\procurement.db

- 引擎：SQLite 3，行工厂 sqlite3.Row（dict-like 访问）

- 连接方式：check_same_thread=False（多线程安全）

- 版本管理：\_migrate_tables() 自动检测和修复表结构

**4.2 核心表结构（共 15 张表）**

**4.2.1 采购垫付表 (purchase)**

id, date, project, handler, payment_method, invoice_status,
reimbursement_status, remark, archived, created_at

**4.2.2 采购物料明细表 (purchase_items)**

id, purchase_id(FK), name, spec, quantity, unit_price, supplier, total

**4.2.3 物料下单表 (packaging_orders)**

id, date, project, supplier, contact, material_name, spec, quantity,
unit_price, total, \... 5环节状态字段

**4.2.4 差旅报销主表 (travel)**

id, date, project, handler, departure, destination, start_date,
end_date, days, reimbursement_status, archived, created_at

**4.2.5 差旅交通明细表 (travel_transport)**

id, travel_id(FK), transport_type, departure, destination, date, amount

**4.2.6 差旅住宿明细表 (travel_hotel)**

id, travel_id(FK), hotel_name, check_in, check_out, nights, amount

**4.2.7 供应商表 (suppliers) \[V1.2\]**

id, name, category, contact, phone, wechat, address, cooperation_status,
remark

**4.2.8 催款记录表 (collection_reminders) \[V1.2\]**

id, supplier, contact, wechat, amount, notify_internal, notify_manager,

**4.2.9 物料台账表 (material_ledger)**

id, year, supplier, material_name, item_no, spec, quantity, unit,
unit_price, total, \...

**4.2.10 项目表 (projects)**

id, name, created_at

**4.2.11 备忘录表 (memos) \[V1.3\]**

id, project, content, handler, status, remark, created_at, updated_at

**4.2.12 报价单产品表 (quotation_products) \[V1.3\]**

id, item_no, product_name, dimensions, material_process, lead_time,

**4.2.13 报价单阶梯表 (quotation_tiers) \[V1.3\]**

id, product_id(FK), min_qty, max_qty（阶梯数量范围，单价留给供方填写）

**4.2.14 报价单需方配置表 (quotation_config) \[V1.3\]**

id, demander_name, contact, phone, address, payment_method,
transport_method, shipping_docs, quotation_requirements

**4.2.15 报价单供方配置表 (quotation_supplier) \[V1.3\]**

id, supplier_name, contact, phone, address, quote_date, validity_period

**4.3 数据库迁移机制**

\_migrate_tables() 方法自动检测并修复表结构：

- 添加缺失列（ALTER TABLE ADD COLUMN）

- 列名重映射（重建表 + 数据迁移，含备份恢复机制）

- 兼容旧版本数据库自动升级

- 每个操作均有 try-except 容错

**六、模块详细说明**

**6.1 仪表盘 (dashboard_page.py) \[V1.3\]**

**6.2 物料下单 (packaging_page.py)**

包材/物料采购订单全流程跟踪，PackagingPage + PackagingForm。

- 5 个可折叠环节板块：物料比价 → 合同跟踪 → 通知厂家 → 生产进度 →
  发货跟进

- 合同状态：待签批 / 已签订 / 已邮寄

- 支持 Excel 导入导出，按项目筛选和归档切换

**6.3 报价单 (quotation_page.py) \[V1.3\]**

产品阶梯定价 + 自动生成 Excel 报价单，QuotationPage + QuotationForm +
配置弹窗。

**6.3.1 功能特点**

- 产品管理：项目号、产品名称、尺寸、材质/工艺、供货周期、发货箱规

- 阶梯价格：每个产品支持多个最低数量 / 最高数量阶梯（单价留给供方填写）

- 需方配置弹窗：需方名称、联系人、电话、地址、付款方式、运输方式等 8 项

- 供方配置弹窗：供应商名称、联系人、电话、地址、报价日期、报价有效期

- Excel
  导出：基于内置模板（产品包装报价单_模板.xlsx），保留所有格式、Logo、合并单元格、打印设置

**6.3.2 导出 Excel 流程**

- 从 \_MEIPASS 加载模板文件，保留原表所有格式和打印设置

- 动态填充需方信息（E5:F11）、供方信息（M5:O10）、条款

- 数据行：解除合并 → 按阶梯数量插入行 → 写入产品数据 → 产品信息跨行合并

- 文件名自动递增：产品包装报价单_YYYYMMDD_001.xlsx

- 使用 openpyxl 直接操作单元格，保留模板原始样式

**6.3.3 模板路径兼容**

> def \_get_resource_path(relative_path): if hasattr(sys,
> \'\_MEIPASS\'): return os.path.join(sys.\_MEIPASS, relative_path)
> return os.path.join(os.path.dirname(\_\_file\_\_), relative_path)

**七、导入导出规范**

**7.1 统一导出接口**

所有页面的导出均调用 Database.export_to_xlsx(filepath, sheet_name,
headers, rows, col_widths) 静态方法。

- 表头：莫兰迪陶土色 #C1816D 背景，白色加粗文字，居中

- 数据行：微软雅黑 10px，隔行浅灰底色 #F7FAFC

- 所有单元格统一细边框

**7.2 报价单导出（特殊处理）**

报价单不使用统一导出接口，而是基于内置 Excel 模板动态填充：

- 从 assets/产品包装报价单_模板.xlsx 加载模板

- 保留原表所有格式：Logo 图片、合并单元格、列宽、行高、打印设置（横向）

- 动态解除合并 → 插入/删除行 → 填入数据 → 产品信息跨行合并

- 需方信息板左侧嵌入同仁堂 Logo 图片（openpyxl Image）

**八、异常处理与日志**

**8.1 全局异常捕获**

- sys.excepthook = \_log_exception → 所有未捕获异常自动写入 error.log

- 日志路径：%USERPROFILE%\\采购管理系统数据\\error.log

- 每次异常写入分隔线 + 完整 traceback

**8.2 表单异常处理**

- \_open_form() 方法使用 try-except 捕获异常并展示 traceback 弹窗

- 确保表单创建失败不会导致程序崩溃

**8.3 数据库迁移容错**

- \_migrate_tables() 中每个 ALTER/REBUILD 操作均用 try-except 包裹

- 支持旧版本数据库平滑升级，单列迁移失败不影响整体

**8.4 托盘容错**

- pystray 导入失败时自动禁用托盘功能

- 托盘图标启动失败时恢复窗口显示（\_do_restore）

**十、技术要点**

**10.1 页面切换机制**

- 所有页面类实现统一接口：\_\_init\_\_(parent, db, colors)

- 切换时先 destroy() 所有子组件，再实例化新页面

- 侧边栏按钮自动高亮当前页面（sidebar_active 配色）

- 设置按钮独立高亮逻辑（\_open_settings 方法）

**10.2 导航栏系统详细设计**

**10.2.1 布局结构**

- 侧边栏 (sidebar)：155px，pack_propagate(False) 固定宽度

- pin_frame：36px 高，置顶按钮右对齐，pack side=top

- nav_area：CTkFrame transparent，fill=both expand=True，pack side=top

- 折叠按钮：pack side=bottom，固定于侧边栏底部

**10.2.2 导航图标**

- 10 个 20×20 PNG 图标（PIL ImageDraw 几何绘制，颜色 #5D4E37，透明背景）

- 通过 CTkImage 加载，存储在 \_nav_icon_images 字典

**10.2.3 紧凑模式（\_toggle_compact）**

- sidebar 宽度 155→46px，按钮 compound=center，仅图标 + tooltip

- 底部按钮切换为展开图标，置顶按钮同步缩小

**10.2.4 悬浮提示**

- \_add_tooltip(widget, text)：绑定 Enter/Leave 事件

- 创建黄色 floating Toplevel（#FFFFCC 背景，solid 边框）

**10.3 窗口置顶（\_toggle_pin）**

- self.attributes(\'-topmost\', True/False) 控制窗口层级

- 📌 未置顶 / 📍 已置顶，置顶时按钮变 primary 色（莫兰迪陶土色）

- 紧凑模式下置顶按钮同步缩小（30×28 → 24×24）

**10.4 系统托盘（pystray）**

- \_minimize_to_tray()：self.withdraw() 隐藏窗口 → 新线程启动托盘图标

- 托盘菜单：显示窗口（default）+ 退出应用

- \_do_restore()：deiconify + lift + focus_force + 短暂置顶闪烁

- \_quit_app()：停止托盘 + 关闭数据库 + destroy + sys.exit

**10.5 注册表开机自启（winreg）**

- 注册表路径：HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run

- 值名：采购助手，REG_SZ 类型

- 打包 exe：写入 exe 绝对路径；开发模式：写入 python.exe + 脚本路径

- SettingsPage.\_\_init\_\_ 中同步注册表状态到 settings 字典

**10.6 数据流**

用户操作 → Page.\_on_click() → db.xxx() → 更新 SQLite → \_do_search() →
\_render()

**10.7 Win10/Win11 适配要点**

- 全局字体统一为 Microsoft YaHei（两个系统均内置）

- 避免使用 Segoe UI（Win10 中文渲染不佳）

- 保持 ctk.set_appearance_mode(\'light\') 亮色主题

- 侧边栏使用 tk.Frame 模拟边框（customtkinter 不支持
  border-directional）

- 窗口居中显示，自适应不同分辨率（1280×800，最小 1100×700）

**10.8 内存管理**

- 页面切换时 destroy() 释放资源

- 表单关闭后自动释放 Toplevel 资源

- 数据库连接使用单例模式（App 级别唯一实例）

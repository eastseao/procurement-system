## 1. 项目概述

### 1.1 系统简介

采购管理系统（"采购助手"）是面向同仁堂健康药业的**桌面端采购管理工具**，覆盖从物料下单、报价、比价、合同制作到催款、垫付、差旅报销的全流程管理。

### 1.2 核心功能矩阵

| 功能模块 | 页面 Key | 说明 |
|---------|----------|------|
| **看板** | `dashboard` | 数据概览：KPI 核心指标（2×2网格）、待办聚合条、最新动态流（30条/8种类型） |
| **物料下单** | `packaging` | 包材下单全生命周期：比价→签批→通知→生产→发货→入库 |
| **报价单生成** | `quotation` | 产品库管理 + 阶梯价格 + 多供应商报价单 Excel 导出 |
| **三方比价** | `compare` | 三家供应商价格对比 + 最终供应商选择 |
| **合同生成** | `contract` | 采购合同自动生成（Word/Excel 模板） |
| **厂家管理** | `supplier` | 供应商档案：分类、合作状态、联系方式 |
| **台账查询** | `query` | 物料台账导入/导出/查询 |
| **BOM 管理** | `product_bom` | 成品 BOM 结构化管理 |
| **催款记录** | `collection` | 应付账款催收跟踪 |
| **采购垫付** | `purchase` | 垫付款记录与报销状态跟踪 |
| **差旅报销** | `travel` | 差旅申请：交通+住宿明细 |
| **备忘录** | `memo` | 待办事项与提醒 |
| **设置** | `settings` | 数据目录选择、主题切换（亮色/暗色）、托盘开关 |

## 2. 技术架构

### 2.1 技术栈

```
┌─────────────────────────────────────────────┐
│                  用户界面层                    │
│         CustomTkinter (CTk) 5.x             │
│    莫兰迪暖色调 / 支持亮色+暗色双主题          │
├─────────────────────────────────────────────┤
│                  业务逻辑层                    │
│     main.py (App) → pages/*.py (各页面)      │
│     database.py (Database 单例)              │
├─────────────────────────────────────────────┤
│                  数据持久层                    │
│            SQLite 3 (procurement.db)          │
│       数据目录: ~/采购管理系统数据/             │
├─────────────────────────────────────────────┤
│                  系统服务层                    │
│   pystray (托盘) / PIL (图标) / openpyxl     │
│   Inno Setup (安装包) / PyInstaller (打包)   │
└─────────────────────────────────────────────┘
```

### 2.2 核心依赖

| 依赖包 | 版本要求 | 用途 |
|--------|---------|------|
| `customtkinter` | ≥ 5.x | UI 框架（现代圆角控件） |
| `Pillow` (PIL) | ≥ 9.x | 图标加载/处理/ICO 解析 |
| `openpyxl` | ≥ 3.x | Excel 导出（台账、报价单、比价单） |
| `pystray` | ≥ 0.x | Windows 系统托盘 |
| `lxml` | ≥ 4.x | 合同模板 XML 解析（可选） |

### 2.3 Python 版本约束

> ⚠️ **必须使用 Python 3.12.10 打包**

原因：Python 3.12.10 环境中已安装 matplotlib 及所有相关依赖。其他版本（如 3.14.3）缺少 matplotlib 会导致 EXE 运行时崩溃。

```bash
# 正确的打包命令
"C:/Users/Administrator/AppData/Local/Programs/Python/Python312/python.exe" -m PyInstaller --clean "采购助手V2.2.0.spec"
```

### 4.2 database.py — 数据库层

**职责**：SQLite 数据库初始化、表结构定义、CRUD 操作、数据迁移、JSON 导入

#### Database 类

```python
class Database:
    def __init__(self, data_dir=None):
        """初始化数据库连接（URI模式+rwc防止PyInstaller权限问题）"""
        self.conn = sqlite3.connect(f"file:{db_uri}?mode=rwc", uri=True,
                                     check_same_thread=False)
        self.conn.execute("PRAGMA journal_mode=MEMORY")  # 内存日志
        self.conn.row_factory = sqlite3.Row               # 字典式访问
```

#### 核心方法分类

| 分类 | 方法 | 说明 |
|------|------|------|
| **项目管理** | `get_projects()`, `add_project()` | 自定义项目列表 |
| **采购垫付** | `save_purchase()`, `update_purchase()`, `get_purchases()` | 含子表 purchase_items |

#### 数据迁移机制

`_migrate_tables()` 在每次启动时执行：
- 检测缺失列 → `ALTER TABLE ADD COLUMN`
- 检测列名不匹配 → 创建备份表 → 数据迁移 → 替换原表
- 兼容旧版本 JSON 文件导入 → `_import_json_data()`

### 4.3 ui_utils.py — UI 工具类

```python
class WheelScrollFrame(ctk.CTkFrame):
    """无可见滚动条的滚动容器，支持鼠标滚轮控制"""
```

**使用场景**：需要自定义滚动行为但不想显示滚动条的场景。

**注意**：内部使用 `tk.Canvas`，默认背景非透明，可能导致视觉上的灰色块问题。对于不需要自定义滚动的场景，推荐直接使用 `CTkScrollableFrame`。

## 9. 开发规范

### 9.1 代码风格

- **编码**: UTF-8
- **语言**: 中文注释 + 中文变量名（业务相关）/ 英文变量名（技术相关）
- **Import 位置**: **必须在文件顶部**（PyInstaller 要求）
- **字符串引号**: 统一双引号 `"`
- **颜色格式**: 只能使用 6 位十六进制 `#RRGGBB`，不支持 8 位 `#RRGGBBAA`

### 9.2 必须遵守的技术规则

详见 `MEMORY.md` 中的完整规则列表，以下是重点摘要：

| 规则 | 说明 |
|------|------|
| Import 位置 | 必须在文件顶部，禁止方法内 import |
| 颜色格式 | 仅支持 `#RRGGBB`，不支持 `"transparent"` 用于 scrollbar 参数 |
| 导航栏折叠 | 展开 content_frame 时必须 `pack(after=underline)` |
| Matplotlib | 必须 `matplotlib.use("TkAgg")` 且在 `import pyplot` 之前 |
| 日期查询 | 月末时间须包含 `23:59:59` |
| 卡片悬停 | 必须记录 `_orig_fg` 以便恢复原始颜色 |
| Sparkline | PIL 使用 `RGBA` 模式 + 透明背景 `(255,255,255,0)` |
| Python 版本 | 打包必须用 Python 3.12.10 |
| excludes 列表 | 必须包含 29 个无用包排除项 |
| openpyxl 模板 | merge_cells 后不能用 None 覆盖左上角；页脚用固定行号 |

## 10. 常见问题排查

### Q1: EXE 运行报 ModuleNotFoundError

**原因**: PyInstaller 未检测到某个 import。
**解决**:
- 本地 `.py` 文件 → 在 spec `datas` 中添加 `('module_name.py', '.')`
- pip 包 → 在 spec `hiddenimports` 中添加 `'package_name'`

### Q2: ValueError: transparency is not allowed

**原因**: 使用了 `"transparent"` 作为 CustomTkinter 不支持的颜色参数。
**解决**: 移除该参数。注意 `fg_color="transparent"` 在部分组件可用，但 `scrollbar_button_color` 等子组件参数不可用。

### Q3: 导航栏展开后子项跑到最下方

**原因**: `pack()` 缺少 `after` 参数。
**解决**: 使用 `pack(after=self.underlines[group], padx=(12,4), pady=(0,4))`

### Q4: 页面中间出现灰色块

**原因**: `WheelScrollFrame` 内部的 `tk.Canvas` 默认背景非透明。
**解决**: 改用 `CTkScrollableFrame`（ctk 原生组件，无灰色块）

### Q5: 任务栏/标题栏图标模糊或不显示

**原因**: ICO 文件分辨率不足，或使用了错误的 API 方式。
**解决**: 统一使用 ×256.ico（含多分辨率），通过 Win32 `LoadImageW(0,0)` 让 Windows 自动选最佳尺寸。

### Q6: EXE 体积过大（>200MB）

**原因**: PyInstaller 保守依赖分析卷入了 ML/AI 包。
**解决**: 确保 spec `excludes` 包含完整的 29 个排除包列表。

### Q7: 暗色模式切换不生效

**原因**: `settings.txt` 中 `theme` 值不是 `light` 或 `dark`。
**解决**: 检查 settings 文件中的 theme 字段。
